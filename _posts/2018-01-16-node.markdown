I can remember a vivid discussion years ago. I can visualize the street and the person that I was talking to. The echo of my voice still bounces in my ears: "using Javascript in the server is a mistake" (probably not in the same terms, you know ;) ). Anyway, server-side JS is a reality, and it's here to stay. And CARTO gave me the chance of learn it for some new, interesting projects, and that's not something that you'd want to dodge. So, let's keep trolling aside and get the most out of Node.js!

Although I've used Javascript in the past (maybe I should dust "[Javascript: The Good Parts](https://www.goodreads.com/book/show/2998152-javascript?ac=1&from_search=true)" off, even though [YDKJS is against it](https://github.com/getify/You-Dont-Know-JS/blob/master/up%20%26%20going/ch3.md)), I'm not up-to-date with the latest shiny language and technology improvements, so there's a lot of things that I should learn. Here are my notes of my path of enlightment.

## Equality

We all know that equality at JS is not _that_ straightworward. Instead of just saying "always use `===`", [Kyle Simpson suggest this](https://github.com/getify/You-Dont-Know-JS/blob/master/up%20%26%20going/ch2.md#equality):

> - If either value (aka side) in a comparison could be the true or false value, avoid == and use ===.
> - If either value in a comparison could be of these specific values (0, "", or [] -- empty array), avoid == and use ===.
> - In all other cases, you're safe to use ==. Not only is it safe, but in many cases it simplifies your code in a way that improves readability.
